# OpenFOAM Case Setup

Every OpenFOAM case is a **directory** with a fixed three-layer structure.
Get the layers consistent first; everything else builds on them.

## The three-layer layout

```
<case>/
├── 0/                     # initial + boundary fields
│   ├── U                  # velocity (always)
│   ├── p                  # pressure (almost always — except buoyant/VOF use p_rgh)
│   ├── k                  # turbulence kinetic energy (RANS k-ε, k-ω SST)
│   ├── epsilon            # k-ε
│   ├── omega              # k-ω SST
│   ├── nut                # turbulent viscosity (RANS)
│   ├── nuTilda            # SpalartAllmaras only
│   ├── T                  # temperature (heat solvers)
│   ├── alpha.water        # phase fraction (VOF — interFoam)
│   └── ...
├── constant/              # mesh + physical properties
│   ├── polyMesh/          # written by blockMesh / snappyHexMesh
│   ├── transportProperties        # ESI: viscosity, density (Newtonian)
│   ├── physicalProperties         # Foundation v11 equivalent
│   ├── turbulenceProperties       # ESI: turbulence model selection
│   ├── momentumTransport          # Foundation v11 equivalent
│   ├── thermophysicalProperties   # heat / compressible solvers
│   └── g                          # gravity vector (buoyant solvers)
└── system/                # numerics + execution control
    ├── controlDict        # start/end time, deltaT, writeInterval, application
    ├── fvSchemes          # discretization schemes
    ├── fvSolution         # linear solvers, algorithm controls (SIMPLE/PISO/PIMPLE)
    ├── blockMeshDict      # mesh definition (when using blockMesh)
    ├── decomposeParDict   # MPI parallel decomposition
    ├── snappyHexMeshDict  # for STL-based meshing (optional)
    └── sampleDict         # post-processing samples (optional, also via -func)
```

The three layers are **coupled**. Changing one usually requires touching
the other two:

- New solver (in `controlDict.application`) → may require different fields
  in `0/`, different properties in `constant/`, different schemes in
  `fvSchemes`.
- New turbulence model (in `turbulenceProperties` or `momentumTransport`)
  → requires the matching turbulence fields in `0/`.
- New BC type on a patch → must be defined in **every** field in `0/` for
  that patch.

## Fork differences (ESI vs Foundation)

| Concept | ESI (v2412 etc.) | Foundation (v11+) |
|---|---|---|
| Transport (single-phase) | `constant/transportProperties` | `constant/physicalProperties` |
| Turbulence selection | `constant/turbulenceProperties` | `constant/momentumTransport` |
| Cavity tutorial path | `incompressible/icoFoam/cavity/cavity` | `incompressibleFluid/cavity` |
| Direct solver binary | `icoFoam`, `simpleFoam` | `foamRun -solver incompressibleFluid` (module-based) |
| `setFields` | available | available, same syntax |

Probe the container to detect the fork:

```bash
which icoFoam simpleFoam buoyantSimpleFoam interFoam dnsFoam 2>&1
ls /opt/openfoam* /usr/lib/openfoam* 2>&1
echo "$WM_PROJECT_VERSION"   # after sourcing etc/bashrc
```

If specific solver binaries exist, you're on ESI; if only `foamRun` exists
or the install lives at `/opt/openfoam11`, you're on Foundation.

## Patch consistency rule

Every patch name **must** appear identically in:
- `constant/polyMesh/boundary` (or be defined by `blockMeshDict`)
- Every field file in `0/`
- Every function object that references that patch

A typo in any one of those three places causes `FOAM FATAL ERROR: cannot
find patch <name>` at solver startup. Pick patch names early and use
exactly the same string everywhere.

## Common patch types

| Type | Purpose |
|---|---|
| `wall` | Solid no-slip / zero-flux boundary |
| `patch` | Generic open boundary (inlet, outlet) |
| `symmetryPlane` | Mirror plane (vector components zero across, scalars symmetric) |
| `empty` | 2D / 1D dimension reduction (front and back of "2D" cases) |
| `cyclic` | Periodic boundary (paired) |
| `wedge` | Axisymmetric wedge case |
| `processor` | Generated by `decomposePar`, never set by hand |

`empty` is critical for 2D cases: a 1-cell-thick slab in z with `empty`
front+back patches makes the solver treat it as 2D. Use a real `wall` or
`symmetryPlane` for the front/back and OpenFOAM treats it as 3D.

## Field file anatomy

Every field file has the same shape:

```
FoamFile
{
    version     2.0;
    format      ascii;
    class       volVectorField;       // or volScalarField
    object      U;                     // matches the filename
}

dimensions      [0 1 -1 0 0 0 0];     // SI units: m·s⁻¹ for velocity

internalField   uniform (0 0 0);      // initial value everywhere

boundaryField
{
    movingWall  { type fixedValue; value uniform (1 0 0); }
    fixedWalls  { type noSlip; }
    frontAndBack { type empty; }
}
```

The dimensions row is `[mass length time temperature moles current
luminosity]`. Common values:

| Field | Dimensions | Meaning |
|---|---|---|
| `U` | `[0 1 -1 0 0 0 0]` | m/s |
| `p` (incompressible) | `[0 2 -2 0 0 0 0]` | m²/s² (kinematic pressure) |
| `p` (compressible) | `[1 -1 -2 0 0 0 0]` | Pa |
| `T` | `[0 0 0 1 0 0 0]` | K |
| `k` | `[0 2 -2 0 0 0 0]` | m²/s² |
| `epsilon` | `[0 2 -3 0 0 0 0]` | m²/s³ |
| `omega` | `[0 0 -1 0 0 0 0]` | 1/s |
| `nut` | `[0 2 -1 0 0 0 0]` | m²/s |
| `alpha.water` | `[0 0 0 0 0 0 0]` | dimensionless |

A wrong dimensions line causes `FOAM FATAL: dimensions mismatch` when
solvers try to combine fields. Get this right per field file.

## controlDict checklist

Minimum viable `controlDict`:

```c++
application     icoFoam;            // or whatever solver
startFrom       startTime;
startTime       0;
stopAt          endTime;
endTime         2.0;                // physical time in seconds
deltaT          0.005;              // initial time step
writeControl    timeStep;
writeInterval   200;
purgeWrite      0;                  // keep all time dirs (set >0 to bound disk)
writeFormat     ascii;
writePrecision  6;
writeCompression off;
timeFormat      general;
timePrecision   6;
runTimeModifiable true;             // re-read controlDict during run
```

For transient with stability concerns, add adaptive timestepping:

```c++
adjustTimeStep  yes;
maxCo           0.9;            // CFL ceiling
maxAlphaCo      0.9;            // VOF only
maxDeltaT       0.01;
```

For steady-state SIMPLE, `endTime` is interpreted as iteration count and
`deltaT` is implicitly 1. Use `convergence` blocks in `fvSolution`
(`SIMPLE { residualControl { ... } }`) to terminate when residuals fall
below thresholds.

## Validating before you run

1. `blockMesh` (or your mesh generator) — should print "End" without errors.
2. `checkMesh -allTopology -allGeometry` — the diagnostic is verbose; what
   matters: `Mesh OK.` at the end. Non-orthogonality > 70° needs `nNonOrthogonalCorrectors > 0`; > 85° needs remeshing.
3. Quick sanity: `ls 0/` lists every required field for your turbulence
   model. Cross-check with `references/field-and-dictionary-matrix.md`.
4. `foamDictionary -entry boundaryField -keywords 0/U` lists patch names —
   compare with `foamDictionary -entry boundary -keywords constant/polyMesh/boundary`. The patch names must agree.

If any of these fail, fix before running the solver. Diverging because
the mesh is bad costs many minutes of wall time you don't need to spend.
