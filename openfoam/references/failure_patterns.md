# OpenFOAM via sim — Failure Patterns

Catalog of failure modes observed across 343 OpenFOAM tutorials, with the
root cause and the minimal fix for each.

## Failure taxonomy

| Category | Meaning | Recovery |
|----------|---------|----------|
| **FAIL_PRECHECK** | Missing input file at startup | Copy/generate the file |
| **FAIL_RUNTIME** | Solver crashed mid-integration | Numerical setup fix |
| **FAIL_WORKFLOW** | Allrun ran but pipeline broken (wrong order, missing tool) | Read Allrun & follow it |
| **FAIL_WORKFLOW_NOPROGRESS** | Allrun ran, solver never reached first timestep within budget | Increase timeout, or split mesh/solver phases |
| **NO_ALLRUN** | No Allrun script — case is a sub-component of a larger workflow | Run from parent dir |
| **SKIP_NP** | numberOfSubdomains exceeds server core limit | Reduce np in decomposeParDict |

## Failure 1: FAIL_PRECHECK — missing input file

### Symptoms
```
--> FOAM FATAL ERROR: (openfoam-2206)
cannot find file "/tmp/foo/0/p"
```
or
```
Cannot read file "/tmp/foo/constant/geometry/curve2.vtk"
```

### Root causes & fixes

**1.1 Missing `0/` directory (most common)**

- **Cause**: Tutorial ships with `0.orig/` instead of `0/`. The standard pattern
  is for `Allrun` to call `restore0Dir` (a shell function from RunFunctions
  that copies `0.orig` → `0`).
- **Fix**:
  ```bash
  [ -d 0.orig ] && cp -r 0.orig 0
  ```
- **Affected**: nearly every tutorial. If you skip Allrun, this is the #1 trap.

**1.2 Missing geometry file**

- **Cause**: Some tutorials reference shared geometry from
  `$FOAM_TUTORIALS/resources/geometry/`. Allrun's first command is the copy.
- **Example**: `mesh/blockMesh/pipe` needs `curve2.vtk.gz`
- **Fix**:
  ```bash
  mkdir -p constant/geometry
  cp $FOAM_TUTORIALS/resources/geometry/curve2.vtk.gz constant/geometry/
  ```

**1.3 Missing chemistry tables**

- **Cause**: chemFoam needs `constant/reactions` and `constant/thermo`
  generated by `chemkinToFoam` from CHEMKIN-format inputs.
- **Example**: `combustion/chemFoam/gri`
- **Fix**:
  ```bash
  chemkinToFoam chemkin/chem.inp chemkin/therm.dat \
                chemkin/transportProperties \
                constant/reactions constant/thermo
  ```

**1.4 Missing initial molecular state**

- **Cause**: MD solvers need `mdInitialise` to populate molecules.
- **Example**: `discreteMethods/molecularDynamics/mdEquilibrationFoam/periodicCubeArgon`
- **Fix**:
  ```bash
  mdInitialise
  ```

**1.5 Chained tutorials**

- **Cause**: `transient` cases that depend on a `steady` precursor having run first.
- **Example**: `incompressible/pimpleFoam/LES/periodicHill/transient` depends
  on `.../steadyState`
- **Fix**: Run the steady case first, then `mapFields` into the transient case.

### Universal fix
**Just run `./Allrun`** — it handles all of these correctly.

## Failure 2: FAIL_RUNTIME — solver crashed

### Symptoms
```
[*] #0  Foam::error::printStack(...)
[*] #1  Foam::sigFpe::sigHandler(int)
Floating point exception (core dumped)
```
or
```
Solving for U
diverged at iteration 152
```

### Root causes

**2.1 Numerical instability from default endTime/deltaT**

- Some tutorials are tuned for a specific Co (Courant number). Modifying
  controlDict (e.g., aggressive sed of endTime) can leave deltaT too large.
- **Diagnosis**: Look for "Courant Number" lines in the log; if Co > 5
  consistently, the timestep is too big.
- **Fix**: Don't modify controlDict. Use the tutorial's own settings.

**2.2 Missing turbulence model wall functions**

- After `cp 0.orig 0`, some tutorials need additional cell-zone setup
  (e.g., setExprBoundaryFields) that's only in Allrun.
- **Fix**: Use Allrun.

**2.3 Genuine tutorial bug or environment mismatch**

- **Examples** (5 cases in our test): vofToLagrangian variants, surfaceMountedCube/fullCase,
  motorBike_leakDetection, foamyHexMesh/mixerVessel.
- These need case-by-case investigation. Often they require specific OpenFOAM
  version features or third-party libs (e.g., gmsh for mesh import).

### Diagnosis checklist
```bash
# Was the solver actually called?
grep "Exec   :" log.<solver>

# Did it reach any timestep?
grep "^Time =" log.<solver> | head -3

# Where did it crash?
grep -A5 "FOAM FATAL\|sigHandler\|stack trace" log.<solver>
```

## Failure 3: FAIL_WORKFLOW — pipeline order wrong

### Symptoms
- Solver never starts
- An Allrun step fails (blockMesh, snappyHexMesh, decomposePar)
- Multi-region cases miss a region directory

### Root causes

**3.1 Skipping Allrun on multi-stage cases**

- **Example**: `basic/chtMultiRegionFoam/2DImplicitCyclic` needs
  `Allmesh` (a separate script) before the solver. Manual `blockMesh`
  doesn't create the per-region meshes.
- **Fix**: `bash Allmesh` then `bash Allrun`, or just `bash Allrun.pre + Allrun`.

**3.2 Wrong Allrun invocation (`bash Allrun` instead of `./Allrun`)**

- See Success Pattern 1 above. The `cd "${0%/*}"` line breaks with `bash Allrun`.
- **Fix**: `chmod +x Allrun && ./Allrun`

**3.3 Sub-case orphan (NO_ALLRUN)**

- Case has `system/decomposeParDict` but no `Allrun` because it's a component
  of a larger multi-case workflow.
- **Examples**: `multiphase/compressibleInterFoam/laminar/waterCooler/fluid`
  (parent case is `waterCooler/`, with separate `fluid/` and `solid/` regions
  managed by the parent Allrun).
- **Fix**: Find the parent directory:
  ```bash
  parent=$(dirname "$case_path")
  while [ ! -f "$FOAM_TUTORIALS/$parent/Allrun" ] && [ "$parent" != "." ]; do
    parent=$(dirname "$parent")
  done
  ```

**3.4 mesh utility produces wrong output**

- snappyHexMesh sometimes hits feature-edge limits that need
  `surfaceFeatureExtract` first.
- **Fix**: Look at Allrun.pre / Allrun for the full mesh sequence.

## Failure 4: FAIL_WORKFLOW_NOPROGRESS — timeout in pre-processing

### Symptoms
- Allrun ran for the full timeout
- No `Time = X` lines in any solver log
- Last log file is `log.snappyHexMesh` or `log.decomposePar`

### Root cause
The mesh generation phase consumed the entire time budget. The solver
itself never started.

### Examples (21 cases in our test)
- **DTCHull, DTCHullMoving** — ship hull with O(M) cells, snappyHexMesh
  takes 5–10 minutes
- **motorBike LES** — fine LES mesh, snappyHexMesh + setHullFvSchemes pipeline
- **lumpedPointMotion bridge/building** — big snappyHexMesh
- **windAroundBuildings** — STL of city geometry

### Fix
1. **Increase timeout proportionally to expected mesh size**:
   ```bash
   case_size=$(du -s constant/triSurface 2>/dev/null | awk '{print $1}')
   if [ "$case_size" -gt 10000 ]; then to=900; fi
   ```
2. **Or split into mesh + solve phases**, each with its own timeout:
   ```bash
   timeout 600 ./Allrun.pre   # mesh only
   timeout 300 ./Allrun        # solver only
   ```

## Failure 5: SKIP_NP — too many cores requested

### Symptoms
- decomposeParDict requests `numberOfSubdomains 128;`
- Server has only 100 cores

### Examples
- 1 case in v2206 tutorials uses np=128

### Fix
Either reduce np in the dict (and re-decompose), or skip:
```bash
np=$(grep -E '^numberOfSubdomains' system/decomposeParDict | head -1 \
     | awk '{print $2}' | tr -d ';')
if [ "$np" -gt 100 ]; then
  # Reduce to 100
  sed -i "s/numberOfSubdomains *$np/numberOfSubdomains 100/" \
      system/decomposeParDict
  # And update method if needed
fi
```

## False positives we hit

These were originally classified as failures, but on investigation were actually
agent/runner issues, not real failures:

| Original FAIL | True cause | Real status |
|---------------|-----------|-------------|
| chemFoam/gri | Skipped chemkinToFoam pre-step | PASS once Allrun is used |
| thermocoupleTestCase | Bad sed corrupted controlDict | PASS once sed removed |
| mdEquilibrationFoam/argon | Skipped mdInitialise pre-step | SLOW_PASS once Allrun is used |
| blockMesh/pipe | Skipped geometry copy | PASS once Allrun is used |
| interFoam/weirOverflow | Mistook bash's `timeout` SIGFPE message for solver FPE | SLOW_PASS (Time=20.76 reached) |

**All 5 of our original "failures" were agent execution errors, not OpenFOAM
or environment problems.** Lesson: trust the tutorial, run Allrun, read the
solver log to judge progress.

## Triage decision tree

```
Did you run ./Allrun?
├── No → use Allrun! (>90% of "failures" disappear)
└── Yes
    ├── exit 0 + "End" in log → PASS
    ├── exit 124/137 (timeout)
    │   ├── log.<solver> shows Time = X > 0 → SLOW_PASS (not a failure)
    │   └── No Time = X anywhere → FAIL_WORKFLOW_NOPROGRESS (need bigger timeout)
    ├── "FOAM FATAL" in any log
    │   ├── "cannot find file" → FAIL_PRECHECK (run pre-step)
    │   ├── "FPE / NaN / diverged" → FAIL_RUNTIME (numerical, real bug)
    │   └── other → FAIL_WORKFLOW (read full log)
    └── Other non-zero exit → FAIL_RUNTIME(rc=N)
```
