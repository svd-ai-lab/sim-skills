# Observation Coupling — Visual Proof

Generated by `tests/execution/mechanical_observation_coupling.py`.

## The experiment

Proves the contract documented in
`base/reference/observation_commands.md`: every `run_python_script`
call mutates Mechanical's in-memory model, and Mechanical's GUI
window (running on the same desktop) repaints immediately, so
`sim screenshot` always reflects the current SDK state.

## The frames

| Stage | What happened | SDK state | Visual state |
|---|---|---|---|
| `01_baseline.png` | Fresh Mechanical launch | `n_analyses=0` | empty outline |
| `02_after_add_static.png` | `Model.AddStaticStructuralAnalysis()` | `n_analyses=1, types=[Static]` | 'Static Structural' appears |
| `03_after_add_modal.png` | `Model.AddModalAnalysis()` | `n_analyses=2` | both in outline |
| `04_final.png` | nothing — just a final capture | stable | stable |

## What this proves

1. `launch(ui_mode='gui')` actually opens a visible Mechanical window.
2. `run_python_script` mutations are reflected in the GUI within one frame.
3. sim's screenshot path (`PIL.ImageGrab`) is independent of the
   gRPC channel, so it works even while `exec` is blocked.
4. `session.summary` (local metadata, no round-trip) stays consistent
   with the round-trip state reported by `run_python_script`.

Reproduce:

```bash
cd E:/simcli/sim-cli
uv run python tests/execution/mechanical_observation_coupling.py
```
